Privacy Policy

Last updated: 2026-04-27

The Technical University of Munich and the Research Group for Applied Education Technologies (AET) take the protection of personal data seriously. Helios processes personal data both when you visit this website and when you use Helios after signing in. Processing is carried out in compliance with applicable data protection regulations, in particular:
  • The General Data Protection Regulation (GDPR)
  • The Bavarian Data Protection Act (BayDSG), which applies to public institutions in Bavaria
  • The Bavarian Higher Education Innovation Act (BayHIG), which defines the tasks of Bavarian universities
  • The Telecommunications Digital Services Data Protection Act (TDDDG), where applicable to technically necessary browser-side storage and comparable technologies
Below, we inform you about the categories of personal data processed in Helios, the purposes and legal bases of the processing, the recipients of the data, the retention periods and the relevant contact points.

What Helios is and who can see what


Helios is operated by AET to manage the development, deployment, and release of AET's own open-source software (e.g. Artemis, Athena, Hephaestus). Helios is not used in any course.


The dashboards Helios builds — including the GitHub usernames and avatars of the developers behind the activity shown — are visible to anyone on the open web without signing in. Signing in is required only to perform write actions (e.g. triggering a deployment, reserving a test environment), and such actions additionally require write permission on the affected GitHub repository.


Purposes and legal bases for processing


The following legal bases apply to the corresponding processing operations in Helios:


Core service (authentication, GitHub synchronisation, dashboards) for TUM members

Art. 6(1)(e) GDPR i.V.m. Art. 2 BayHIG (tasks of Bavarian universities — research, teaching, transfer; AET is a research chair operating chair-level development and release infrastructure for the chair's own software, which falls under research and transfer) i.V.m. Art. 4 BayDSG (a Bavarian public body may process personal data when this is necessary to carry out its tasks).


Core service for non-TUM contributors

Art. 6(1)(b) GDPR. External open-source contributors who sign in to Helios with their GitHub account do so on their own request in order to use the service; the processing necessary to provide the service to them is based on this request.


Optional notification email and notification preferences

Art. 6(1)(a) GDPR (consent). Providing a notification email address and choosing notification preferences in the Helios settings page is an opt-in by affirmative action. Consent can be withdrawn at any time with effect for the future, without affecting the lawfulness of processing carried out before the withdrawal (Art. 7(3) GDPR).


Server access logs and self-hosted Sentry error telemetry

Art. 6(1)(f) GDPR (legitimate interests). AET has a legitimate interest in operating Helios securely and reliably, in detecting and diagnosing errors, and in defending against attacks. The balancing of interests is in favour of processing because the data is technical rather than behavioural, is not used for profiling, is retained only briefly (see retention table below) and stays inside TUM-operated infrastructure.


Categories of personal data processed


When you sign in, Helios authenticates you via a self-hosted Keycloak federated to GitHub OAuth and stores the following data associated with your account:


  • GitHub login, display name, avatar URL and profile URL
  • Email address provided by GitHub
  • Optional GitHub profile fields: bio, company, blog, free-text location (from the GitHub profile, not GPS), follower and following counts
  • Optional notification email address you enter in the Helios settings page
  • Notification preferences (currently for the events DEPLOYMENT_FAILED, LOCK_EXPIRED and LOCK_UNLOCKED)
  • Technical UI state stored in your browser (e.g. expanded or collapsed state of the sidebar)

For developers who have not signed in to Helios but whose activity in AET-maintained GitHub repositories is mirrored in Helios's dashboards, the personal data shown originates from the developer's own public GitHub account, accessed via the GitHub API (Art. 14 GDPR).


Logging


When Helios is accessed, the web application and backend services process technical information in log files in order to provide the service securely and reliably. This may include the following information:


  • IP address of the requesting computer
  • Date and time of access
  • Name, URL and transferred data volume of the retrieved file
  • Access status (requested file transferred, not found, etc.)
  • Identification data of the browser and operating system used (if transmitted by the requesting web browser)
  • Web page from which access was made (if transmitted by the requesting web browser)

The processing of the data in this log file takes place as follows:


  • The log entries are continuously and automatically evaluated in order to detect attacks, errors and misuse and to react accordingly.
  • In individual cases, i.e. in the case of reported disruptions, errors and security incidents, a manual analysis is carried out.
  • The data is not used to create personal user profiles.

Recipients


The following recipients receive personal data in the course of operating Helios:


  • Anyone on the open web — the dashboards Helios builds (workflow runs, deployments, environments, environment-lock history, branches, commits, pull requests, releases, issues, test results) include the GitHub-derived developer attribution attached to each item and are served without authentication via every GET endpoint under /api/**.
  • GitHub, Inc. — Helios reads repository activity from GitHub via the GitHub API and also sends deployment-trigger actions back to GitHub Actions on behalf of a signed-in user when that user triggers a deployment.
  • Self-hosted Keycloak operated within TUM — authentication.
  • Self-hosted Sentry at sentry.ase.in.tum.de, operated within TUM — error telemetry. No external Sentry SaaS is used.
  • TUM SMTP / AET mail relay operated within TUM — delivery of notification emails.

Transfer to a third country


GitHub, Inc. is based in the USA. The transfer of personal data to GitHub is covered by GitHub's own certification under the EU–U.S. Data Privacy Framework (Art. 45 GDPR) and by Standard Contractual Clauses (Art. 46(2)(c) GDPR; Commission Implementing Decision 2021/914), as set out in the GitHub Data Protection Agreement, which applies to every github.com user. No personal data is transferred to any other recipient outside the European Economic Area.


Retention periods


  • Account and profile data: for as long as the account exists; deleted on request (see “Your rights” below).
  • Workflow run records (database): a nightly cleanup task at 01:00 keeps the two newest PROCESSED runs per repository and branch and deletes runs in state FAILED or with no terminal state that are older than five days.
  • Workflow run logs (filesystem): a nightly cleanup task at 02:00 deletes log files older than one day.
  • Self-hosted Sentry error telemetry: retained for at most 30 days.
  • Database backups: daily backups with a retention of 7 days.
  • Server access logs: standard log rotation.

Unless statutory retention obligations require otherwise, personal data is stored only for as long as necessary to operate Helios and fulfil the respective processing purpose.


Technical operation and cookies


The technical operation of Helios is carried out within the environment of the Technical University of Munich and the Research Group for Applied Education Technologies. The Helios web servers are operated by AET itself at Boltzmannstraße 3, 85748 Garching b. München.


Helios uses technically necessary browser-side storage and comparable session mechanisms to provide the application. This includes, for example, storing UI state in the browser. If you disable such storage mechanisms, parts of the application may no longer work correctly.


No automated decision-making


Helios does not perform automated decision-making, including profiling, within the meaning of Art. 22 GDPR.


Notification email (consent)


You may optionally enter a notification email address and choose notification preferences on the Helios settings page. This processing is based on your consent (Art. 6(1)(a) GDPR). You can change or remove the notification email address and your notification preferences at any time on the same settings page. Withdrawing consent does not affect the lawfulness of processing carried out before the withdrawal (Art. 7(3) GDPR).


Correspondence by email


If you contact us by email, your email address and the content of your message are processed solely for the purpose of handling your enquiry. Please note that data transmission over the internet can have security gaps; complete protection of data from access by third parties is not possible.


Your rights


You have the following rights with respect to the personal data Helios processes about you:


  • Right of access (Art. 15 GDPR)
  • Right to rectification (Art. 16 GDPR)
  • Right to erasure (Art. 17 GDPR)
  • Right to restriction of processing (Art. 18 GDPR)
  • Right to data portability (Art. 20 GDPR)
  • Right to object (Art. 21 GDPR) — in particular against processing based on legitimate interests (Art. 6(1)(f) GDPR), namely server access logs and self-hosted Sentry error telemetry
  • Right to withdraw consent (Art. 7(3) GDPR) — in particular with respect to the optional notification email address and notification preferences; withdrawal does not affect the lawfulness of processing carried out before the withdrawal
  • Right to lodge a complaint with a supervisory authority (Art. 77 GDPR; see below)

To exercise these rights, you can contact the AET Administrators at ls1.admin@in.tum.de or the official TUM data protection officer at beauftragter@datenschutz.tum.de.

SSL/TLS encryption


For security reasons and to protect the transmission of confidential content, Helios uses SSL/TLS encryption. This means that data transmitted via this website cannot be read by third parties during transport. You can recognise an encrypted connection by the “https://” address in your browser and by the lock symbol in the browser interface.
